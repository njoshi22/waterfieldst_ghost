1 . Getting Started

For the purposes of this documentation, it is assumed that you have Ghost up and running. If not, please visit support.ghost.org and follow the guide to install and configure Ghost.

Please make sure you are running the latest version of Ghost before installing & using Quartz.

2. Installation

  1. Navigate to the directory on your server where Ghost is installed.
  2. Once there, navigate to the /content/themes/ directory.
  3. Upload the theme files to this directory.
  4. Restart Ghost.
  5. Log into your Ghost Admin.
  6. Navigate to Settings > General and select Quartz in the theme options dropdown menu.
  7. Click 'Save'.

Note: If you're on using Ghost(Pro), please follow these instructions on how to install a theme.

Public API

This theme uses the Public API feature in Ghost, currently this feature is in beta and you will have to enable it in order to use all the theme features.

  1. Go to Settings > Labs in the Ghost admin area.
  2. Scroll to the bottom of the page
  3. Under Enable Beta Features, tick the checkbox next to Public API.

3. Upgrading

If you have made any changes to Quartz's template or SCSS/CSS files, you will need to re-integrate your custom theme changes.

Always perform a backup of the theme files before upgrading

  1. Download the latest version of Quartz from your purchases page on Creative Market.
  2. Navigate to the folder on your server where Ghost is installed.
  3. Once there, navigate to the /content/themes/ folder.
  4. Upload and replace all of the older theme files with the newer ones.

Once you update your theme, you will want to restore your changes to the theme-options.js file.

4. Settings

In Settings > General several options will alter the content in Quartz:

  • Blog Logo - Appears as the first item the header. We recommend a maximum size of 240 Ã— 30 pixels.
  • Blog Title - The blog title will be displayed if no blog logo has been uploaded.
  • Blog Description - Appears as your blog description in the search engines.
  • Blog Cover - This theme does not use or display the cover photo.
  • Posts per page - This is how many posts are displayed per page. We recommend setting this to an 10 for Quartz to display an equal number of items per page.

5. Customization

If you're comfortable with HTML & CSS, you can make any changes to the theme by editing Quartz's template files.

Please be aware that when Quartz is updated in the future, you will need to re-integrate your custom theme changes.

The CSS files were created using SASS. If you are familiar with SASS, you can easily customize Quartz.

Using CSS

You can add your own custom CSS by using the Code Injection feature in Ghost.

If you have existing CSS in a custom.css stylesheet, you can cut & paste your CSS into the Blog Header using Code Injection.

Note: Don't forget to wrap your all your CSS in <style> tags.

Using SASS

You will find the following SCSS files in /assets/scss/ directory:

  • screen.scss - This is the main CSS stylesheet. It imports the mixins.scss and variables.scss files.
  • mixins.scss - This contains all the CSS declarations that Quartz reuses throughout the theme.
  • variables.scss - This contains all the variables that you can tweak to customize the way Quartz looks.
  • normalize.scss - This contains Normalize.css, which is a customisable CSS file that makes browsers render all elements more consistently and in line with modern standards.

Note: You will need a SASS preprocessor, such as Grunt to use the SCSS files.

Navigation

You can customize the navigation menu by using the Navigation builder in your Ghost admin area.

  1. Go to Settings > Navigation in your Ghost admin area.
  2. Add, remove and/or reorder navigation items on the right.
  3. Click save.

Social Media

Add your own social media links to the header by editing the theme-options.hbs file located in the partials > js folder.

  1. Navigate to the partials > js folder on your server.
  2. Open the theme-options.js file in your favorite text editor.
  3. Find the Social Media section

  4. Add your social media links (between the quotation marks) to the corresponding social media variables.

    Facebook Example:

     var facebook_link = "http://www.facebook.com/my_facebook_username";

  5. Upload the changed theme-options.js file to the partials > js folder.

Note: If you don't want a specific social media icon to appear, just leave variable blank.

var facebook_link = "";

Note: If you wish to show RSS icon, change the rss_link value to true.

var rss_link = true;

Post Cover Images

You can add a post cover image to your posts by adding one in the post settings. This post cover image will be displayed along with the title of the post on the homepage and in the post itself.

  1. Login to the Ghost Admin area.
  2. Create or edit a post.
  3. Click the post settings icon, this is located next to the publish/update post button.
  4. Click "Add post image.
  5. Save/update the post.

Featured Posts

Any posts that you set as featured will appear in the slider on the home page.

We recommend setting a minimum of 4 posts as featured in order for the slider to look the best.

Featured Post Slider

You can customize many options for the featured post slider by editing the theme-options.js file located in the assets > js folder:

  1. Navigate to the partials > js folder on your server.
  2. Open the theme-options.js file in your favorite text editor.
  3. Find the Featured Posts Slider section.

  4. Edit any of the following slider variables, below are the descriptions and options:

    slider_auto_play_speed: Time between slide transitions, in milliseconds. If you don't want the slider to auto play, just set the value to 0.

    This value will set the slide transition speed to 4 seconds:

    var slider_auto_play_speed = 4000;

    This value will disable the slider autoplay:

    var slider_auto_play_speed = 0;

    slider_wrap_around: This will make the slider wrap-around to the other oend for infinite scrolling, set as true or false.

    slider_draggable: Enables dragging and flicking, set as true or false

    slider_nav: Display slider navigation (the arrows), set as true or false

This feature requires you to enable the Public API beta:

  1. Go to Settings > Labs in the Ghost admin area.
  2. Scroll to the bottom of the page
  3. Under Enable Beta Features, tick the checkbox next to Public API.

Sidebar

The sidebar will showcase your featured posts, latest posts, author information and next/previous posts.

By default, only 4 of your featured posts and latest posts will be displayed in the sidebar.

This feature requires you to enable the Public API beta:

  1. Go to Settings > Labs in the Ghost admin area.
  2. Scroll to the bottom of the page
  3. Under Enable Beta Features, tick the checkbox next to Public API.

Recommended Stories

At the end of each post, 3 recommended stories will be shown to the user. These recommended stories are posts written by the author of the current post.

This feature requires you to enable the Public API beta:

  1. Go to Settings > Labs in the Ghost admin area.
  2. Scroll to the bottom of the page
  3. Under Enable Beta Features, tick the checkbox next to Public API.

Pull Quotes

Pull quotes are a small selection of text that are "pulled out" and quoted.

It's easy to use pull quotes, all you have to do in place the following HTML code where you want the pull quote to be shown:

<blockquote class="pullquote">
Please replace this text with your own quote!
</blockquote>

Don't forget to replace the text inbetween the <blockquote> HTML tag with your own quote!

Comments

You can enable Disqus comments on your blog by adding your Disqus shortname to the theme-options.js file located in the assets > js folder:

  1. Add your site to your Disqus account and setup a unique Disqus URL. The unique shortname you pick for your Disqus URL will be important to remember!
  2. Navigate to the assets > js folder on your server.

  3. Open the theme-options.js file and add your unique shortname to the disqus variable.

    Example:

     var disqus = "DISQUS_SHORTNAME_HERE";

  4. Save/update the theme-options.js file.

Note: If you don't want comments on your blog just leave the disqus variable blank.

var disqus = "";

Image Lightbox

By default Quartz will automatically add an image lightbox to all images within a post. You can disable the automatic image lightbox by editing the theme-options.js file located in the partials > js folder.

  1. Navigate to the partials > js folder on your server.
  2. Open the theme-options.js file in your favorite text editor.

  3. Find the lightbox variable and change the value from true to false.

    Example:

     var lightbox = false;

  4. Upload the changed theme-options.js file to the partials > js folder.

Note: If you wish to enable the automatic image lightbox again just change the false value to true.

var lightbox = true;

Infinite Scroll

By default infinite scroll will be disabled. You can enable infinite scroll by editing the theme-options.js file located in the partials > js folder.

  1. Navigate to the partials > js folder on your server.
  2. Open the theme-options.js file in your favorite text editor.

  3. Find the lightbox variable and insert the text (between the quotation marks) scroll or click. Here are the differences between the two options:

    Scroll will load new posts while scrolling:

     var infinite_scroll = "scroll";

    Click will load new posts with a click of a button:

     var infinite_scroll = "click";

  4. Upload the changed theme-options.js file to the partials > js folder.

Note: If you wish to disable infinite scroll just change the infinite_scroll variable blank.

var infinite_scroll = "";
